# Contribuyendo

Este repositorio se usa como base para el TP "Gastos Grupales". La idea es mantenerlo prolijo y con cambios chicos, claros y faciles de revisar.

## Criterios

- priorizar cambios enfocados en el dominio del TP
- evitar sumar mas codigo generico del template original
- mantener consistencia entre backend, frontend y documentacion
- si se cambia comportamiento, actualizar o agregar tests cuando corresponda

## Flujo recomendado

1. definir el objetivo del cambio
2. tocar solo los archivos necesarios
3. verificar que el proyecto siga levantando
4. dejar notas o TODOs si algo queda como paso siguiente

## Antes de abrir un PR

- correr las validaciones relevantes
- revisar textos visibles para que usen el lenguaje del TP
- evitar dejar referencias al template original, assets demo o docs heredadas
